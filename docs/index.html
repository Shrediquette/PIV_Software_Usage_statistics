Methodology ▼

Conflict of interest, stated openly: This tool was built by William Thielicke, the author of PIVlab, and PIVlab ranks first below. The same metric is applied to every package — no rule favours PIVlab — and the complete source code is public. Where a query could be made artificially large it was deliberately tightened, including for competitors. Full rationale and a reusable recipe: METHODOLOGY.md.
Unified methodology (fair, objective comparison): Every software package is counted with the same metric:
  • Exact-phrase full-text search via OpenAlex default.search: software names are searched as quoted exact phrases in the full text (title + abstract + body where indexed). Quotes prevent stemming false-positives.
  • "particle image velocimetry" required: this phrase must appear literally in the text. Guarantees PIV context without relying on ML tags. Eliminates false positives from multi-purpose names (e.g. DaVis, TSI, GPIV).
  • Spelling variants: alternate forms are OR-combined (e.g. "OpenPIV" OR "open piv").
  • Commercial software: the company name must also appear in the text (e.g. "Dantec" AND "Dynamic Studio").
Example filter: default.search:"LaVision" AND "DaVis" AND "particle image velocimetry",publication_year:2010-2025
Known limitation: open-source tools (PIVlab, OpenPIV, …) are almost always named explicitly by authors; commercial tools are sometimes described only as "commercial software" without naming them. This means commercial tools may still be undercounted relative to their true usage, even with full-text search.
Note: OpenAlex covers ~96% of the literature. Only fully completed years (2010–2025) are shown.
